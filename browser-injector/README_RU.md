## 📖 Что такое App-Bound Encryption?

В последних версиях Chromium (начиная с 127+) был введен механизм **App-Bound Encryption**. Он защищает ключи шифрования (Master Key) с помощью COM-сервиса, который проверяет идентичность вызывающего процесса. Это означает, что обычные "стилеры", работающие как сторонние процессы, больше не могут просто прочитать `Local State` и расшифровать куки или пароли через DPAPI.

**BrowserInjector** обходит эту защиту, внедряя полезную нагрузку непосредственно в контекст браузера и используя легитимные COM-интерфейсы браузера для дешифровки.

## ✨ Основные возможности

- **🚀 Обход ABE (App-Bound Encryption)**: Использование COM-интерфейсов `IElevator` и `IElevator2` для получения мастер-ключа.
- **🌐 Поддержка 20+ браузеров**: Включая Chrome (Stable, Beta, Dev, Canary), Microsoft Edge, Brave, Opera (Stable, GX), Vivaldi, Yandex, Avast Browser и многие другие.
- **🛡️ Обход EDR/AV**:
  - Использование **прямых системных вызовов (Direct Syscalls)** для обхода хуков в User-mode.
  - Динамическое разрешение SSN (System Service Number).
  - Шифрование полезной нагрузки (DLL) алгоритмом **ChaCha20**.
  - Инъекция в приостановленный процесс (Suspended Process Injection).
- **📊 Обширная экстракция**: Извлечение Cookies, паролей, данных банковских карт и автозаполнения.
- **🔍 Fingerprinting**: Сбор детальной информации о системе и браузере.
- **💻 Кросс-архитектурность**: Поддержка систем **x64** и **ARM64**.

## 🏗️ Архитектура проекта

Проект разделен на несколько ключевых компонентов:

1.  **Injector (`browser_injector.exe`)**:
    *   Обнаруживает установленные браузеры.
    *   Завершает работающие процессы (опционально).
    *   Создает новый процесс браузера в приостановленном состоянии.
    *   Внедряет зашифрованную DLL (Payload).
    *   Управляет связью через Named Pipes.
2.  **Payload (`chrome_decrypt.dll`)**:
    *   Выполняется внутри контекста браузера.
    *   Взаимодействует с COM-сервисом браузера для дешифровки ключей.
    *   Читает SQLite базы данных профилей.
    *   Передает результаты обратно инжектору.
3.  **Encryptor (`tools/encryptor.cpp`)**:
    *   Утилита для шифрования Payload DLL перед сборкой основного инжектора.

## 🚀 Использование

Запустите скомпилированный файл из командной строки:

```bash
browser_injector.exe [опции] <имя_браузера | all>
```

### Примеры:
- Извлечь данные из всех найденных браузеров:
  ```bash
  browser_injector.exe all
  ```
- Извлечь данные из Chrome с подробным выводом:
  ```bash
  browser_injector.exe chrome -v
  ```
- Собрать fingerprint и принудительно завершить браузер перед работой:
  ```bash
  browser_injector.exe edge -f -k
  ```

### Доступные опции:
| Опция | Описание |
| :--- | :--- |
| `-v`, `--verbose` | Показать подробный лог выполнения. |
| `-f`, `--fingerprint` | Собрать информацию о системе. |
| `-k`, `--kill` | Завершить процессы браузера перед началом работы. |
| `-o`, `--output-path` | Указать путь для сохранения результатов. |

## 🛡️ Технические детали (EDR Evasion)

Инструмент использует продвинутые техники для скрытия своей активности:
- **Syscall Trampolines**: Вместо прямого вызова функций API, проект использует кастомные ассемблерные вставки для выполнения syscall'ов, что делает невозможным их перехват стандартными EDR.
- **Embedded Encrypted Payload**: Основная логика дешифровки хранится в зашифрованном виде внутри инжектора и расшифровывается только в памяти целевого процесса.
- **No CRT dependency in critical parts**: Минимизация использования стандартных библиотек для уменьшения сигнатурного профиля.
